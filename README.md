# Response Refiner

Response Refiner 是一个 SillyTavern 文本处理插件，用于手动润色消息、修正 AI 回复格式，并在回复被截断或标签缺失时按规则补完。

## 功能

- 功能1：润色 AI 回复和用户输入。
- 功能2：格式检查和补全修正，仅处理正文以外的其他标签。
- 功能3：回复补完，参考需要补完回复的上一条用户消息，可额外参考思维链。
- 支持“执行已选”按钮，按脚本判断分步执行，最多消耗三次请求。
- 可在设置中开启生成状态观察窗口；OpenAI 兼容接口优先使用流式输出，Gemini/Claude 不支持时降级为等待完整响应，并在长时间无新状态或新输出时显示等待心跳。
- 处理结果先预览，确认后再替换，可恢复原文；替换后再次执行会以当前消息正文作为新输入，不复用旧候选状态。

## 连接方式

插件支持直接连接（OpenAI 兼容）、OpenRouter、Gemini、Claude、OpenAI、DeepSeek。

OpenRouter、OpenAI、DeepSeek 和直接连接使用 OpenAI 兼容格式。Gemini 和 Claude 使用各自原生接口格式。

## 设置说明

### 连接设置

选择连接方式，填写接口地址、API Key，然后刷新模型列表或手动输入模型名称。配置完成后可测试连接。旧版本遗留的反代连接设置会自动迁移到“直接连接（OpenAI 兼容）”。

### 功能1：润色

润色设置包含 AI 回复润色提示词、用户输入润色提示词、八股文禁用字符/短语和润色捕获正则。

正文规则的开始/结束标签必须和润色捕获正则识别出的简单成对标签一致，例如：

```text
<content>([\s\S]*?)</content>
```

如果正则过复杂，插件不会乱改规则，会提示用户手动修正。润色结果会先作为候选展示，点击替换后才写回消息，并可通过恢复按钮回到替换前原文。

### 功能2：格式检查和补全修正

格式规则由多组成对标签组成。必须保留一条“正文”规则，但功能2不会处理正文标签。

功能2只处理其他标签。请求的 user 消息会先放需要判断、检查、补完的格式标签规则，再放完整原文、当前标签内容 JSON 和严格 JSON 输出约束。模型必须只返回 JSON 形式的标签内替换文本；脚本按 JSON 回填对应标签内部，并按规则顺序整理标签块。JSON 解析失败或返回空字符串清空标签时会报错，不会静默写入半成品结果。

### 功能3：回复补完

回复补完仅对 AI 回复生效。补完上下文改为：

1. user 消息最开头放全部格式标签规则。
2. 然后放完整原文、上一条用户消息、捕获到的思维链、未闭合/缺失标签判断和补完位置说明。
3. 最后强调只输出需要追加到当前回复末尾的内容。

生成后脚本直接把模型返回的追加片段拼接回原文，仅保留正文未闭合时的最小闭合兜底。

### 自动提取格式规则

自动提取普通标签时会让 AI 生成规则建议。若填写思维链捕获正则，会额外让 AI 分析已捕获思维链中的步骤、边界特征、标签/段落结构和可复用格式规则，并输出结构化规则。结构化分析失败时会生成带“兜底草案”标记的规则，需人工复核后再追加。

### 生成参数与状态窗口

可开启“生成过程中显示状态与输出观察窗口”。该窗口只用于观察当前生成状态和流式文本，不会边生成边修改聊天消息。发送提示词后若长时间没有新状态或新输出，窗口会按固定间隔提示已等待时长、当前阶段和可能原因；该提示不代表模型真实进度。完成后仍需要在候选结果预览中确认替换。

### 执行已选

“执行设置中已勾选的功能”不是简单拼接提示词，而是脚本先判断标签状态，再按需执行：

1. 若启用回复补完且存在缺失标签或正文标签未闭合，先执行功能3。
2. 若启用格式检查，执行功能2，只修正非正文标签。
3. 若启用润色，执行功能1，只润色正文标签捕获内容。

因此该按钮最多会消耗三次请求。

## 消息按钮

消息下方会显示可用按钮：润色、格式检查、回复补完、执行已选、停止、替换、恢复。用户输入只显示可用的润色相关操作。

## 安全说明

- API Key 和接口地址只保存在本地 SillyTavern 设置中。
- 预览和状态窗口内容使用文本方式渲染，避免将模型输出作为 HTML 插入页面。

## 文件

- `index.js`：插件逻辑、连接适配、功能管线、流式状态窗口和消息按钮。
- `settings.html`：插件设置界面。
- `style.css`：插件样式。
- `manifest.json`：SillyTavern 扩展清单。
