# Response Refiner

Response Refiner 是一个 SillyTavern 文本处理插件，用于手动润色消息、修正 AI 回复格式，并在回复被截断时按格式规则补完。

## 功能

- 手动润色 AI 回复和用户输入。
- 润色时可通过提示词替换或移除指定八股文字符/短语。
- AI 回复可启用正则过滤，只处理最后一个匹配片段。
- AI 回复可执行格式检查和补全修正。
- AI 回复可执行回复补完，并按已配置格式规则续写。
- 支持“执行已选”按钮，按设置中启用的功能顺序处理。
- 处理结果先预览，确认后再替换，可恢复原文。

## 连接方式

插件支持 7 种连接方式：

1. Cloudflare Worker 反代
2. 直接连接（OpenAI 兼容）
3. OpenRouter
4. Gemini
5. Claude
6. OpenAI
7. DeepSeek

每种连接方式都包含：

- 接口地址
- API Key
- 模型列表刷新
- 模型下拉选择
- 手动模型名称兜底
- 测试连接

OpenRouter、OpenAI、DeepSeek、直接连接和 Cloudflare Worker 使用 OpenAI 兼容格式。Gemini 和 Claude 使用各自原生接口格式。

## 使用方法

### 1. 配置连接

在插件设置中选择连接方式，填写接口地址、API Key，然后点击刷新模型列表或手动输入模型名称。配置完成后可点击测试连接。

### 2. 配置润色

润色设置包含：

- AI 回复润色提示词
- 用户输入润色提示词
- 八股文禁用字符/短语
- AI 回复正则过滤

八股文内容使用逗号分隔。该功能不提供独立按钮，而是在润色提示词中要求模型对原文和输出进行替换或移除。

### 3. 配置格式规则

格式检查规则由多组成对标签组成，每组包含：

- 是否启用
- 名称
- 开始标签
- 结束标签
- 标签内提示词
- 标签内容模板

规则顺序可通过上移、下移调整。格式检查会按顺序修正标签缺失、顺序错误、闭合错误和标签内容不符合模板的问题。

### 4. 配置回复补完

回复补完仅对 AI 回复生效，并配合格式规则工作。

可设置“大纲捕获正则”。如果正则匹配到捕获组，插件会使用最后一个捕获结果作为大纲上下文；如果没有匹配，则使用用户最后一次输入作为补完上下文。

回复补完提示词应强调：只输出续写部分，不重复已有内容，并遵守格式标签规则。

### 5. 消息按钮

消息下方会显示可用按钮：

- 润色
- 格式检查
- 回复补完
- 执行已选
- 替换
- 恢复

用户输入只显示可用的润色相关操作。格式检查和回复补完只对 AI 回复有效。

## 安全说明

- API Key 和接口地址只保存在本地 SillyTavern 设置中。
- 预览内容使用文本方式渲染，避免将模型输出作为 HTML 插入页面。

## 文件

- `index.js`：插件逻辑、连接适配、功能管线和消息按钮。
- `settings.html`：插件设置界面。
- `style.css`：插件样式。
- `manifest.json`：SillyTavern 扩展清单。
